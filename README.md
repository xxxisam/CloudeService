Клиент серверное приложение - аналог облачного хранилища.

Описание: Клиент-серверное приложение, ориентированное на хранение файлов на приложении сервер. Клиент представляет из себя страничку в браузере, через которую пользователь авторизуется и получает доступ к управлению хранимыми данными(файлами, документами) на сервере: загрузка, хранение и скачивание с сервера. Каждому пользователю выделяется некоторое количество памяти для хранения файлов. Изменять это количество может пользователь-администратор. При подключении используется защищенное соединение и digest-авторизация пользователей. Сервер хранит базу данных авторизации.

План реализации:

    Изучить работу библиотек Boost::ASIO и Boost::Beast - основных библиотек используемых в проекте.
    Создать некоторый класс-Сервер и класс-Клиент. Произвести подключение между ними.
    Изучить тему безопастности в сети. Темы защищенного и digest-соединения.
    Создать класс User который будет хранить информацию о данных пользователя. Например, файлы этого конкретного пользователя.
    Создать некоторый класс, который будет отвечать за хранение файлов на сервере, возможно предусмотреть создание структуры, которая будет хранить некоторую информацию о файле и указатели на пользователя, котором данный файл принадлежит.
    Создать метод/функцию которая будет передавать файлы с клиента на сервер и обратно. Также необходимо проверять целостность полученного файла, обработать возможные случаи повторной загрузке уже существующего файла на сервер одним пользователем(Предупредить что файл уже существует и предложить перезаписать его) или случай, когда другой пользователь загружает файл который уже лежит на сервере, но у другого пользователя(Просто вернуть ссылку на этот файл другому пользователю).
    Повторить JS и веб составляющую для последующей интеграции веб-документа(HTML, CSS, JS) в приложение
    Сверстать некоторые веб-страницы состоящие из: Авторизации и Списка файлов(У обычного пользователя); Авторизации, Списка файлов и некоторой админ-панели(у администратора). Админ панель должна представлять из себя список пользователей и позволять изменять размер хранилища пользователей.
    Интегрировать Веб-составляющую и бэкенд в виде сервера в единое приложение.
    Проверить работоспособность приложения. Авторизации, загрузки файлов и их скачивании с сервера.
    Найти способы расширения и добавления иного полезного функционала в приложение.
