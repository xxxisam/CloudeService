

Клиент серверное приложение. 

Описание:

    1. Сервер принимает, хранит(После выключеия, например) и отправляет файлы на клиент пользователю.

    2. С клиента

        2.1 Обычный пользователь регистрируется, получает некоторое количество свободного места на сервере, может отправлять, хранить и получать свои файлы.

        2.2 Сервером может управлять администратор. Он может увеличивать размер свободного места пользователя.
   




  
  План реализации:

1. Реализовать подключение клиента к серверу. 
	1. Необходимо вывести на клиенте инфо, о том что подключение удачно или не удачно. На сервере при удачном подключении вывести ip, откуда было подключение.

2. Реализовать передачу файлов. 
	1. Организация места на сервере.
	2. Отправление с клиента на сервер. 
	3. Проверить целостность полученного файла.
	4. Отправка с сервера на клиент.
3. Реализовать авторизацию пользователей. Разделить пользователей и admin.
	1. Организация проверки данных. Проверка того, кто подключается: Пользователь или admin.
	2. Данные вводятся на клиенте и отправляются на сервер для валидации.
