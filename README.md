

Клиент серверное приложение - аналог облачного хранилища. 

Описание:

    1. Сервер принимает, хранит(После выключеия, например) и отправляет файлы на клиент пользователю.

    2. С клиента

        2.1 Обычный пользователь регистрируется, получает некоторое количество свободного места на сервере, может отправлять, хранить, просматривать и получать свои файлы.

        2.2 Сервером может управлять администратор. Он может изменять размер свободного места пользователя.
	3. При подключении используется защищенное соединение и digest авторизация пользователей.
		3.1 На сервере хранится некоторая база данных с данными авторизации пользователей.
   




  
  План реализации:

1. Реализовать подключение клиента к серверу. 
	1. Отправить с сервера строку об удачном или не удачном подключении.

2. Реализовать передачу файлов. 
	1. Организация места на сервере.
	2. Отправление файла с клиента на сервер. 
	3. Проверить целостность полученного файла.
	4. Получение файла пользователем с сервера на клиент.
3. Реализовать авторизацию пользователей с разными уровнями доступа к серверу.(Пользователи и админы)
